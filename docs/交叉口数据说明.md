# 真实交通交叉口数据说明

> 数据来源：`traffic_full_20260521_112701.sql`（5.0GB）
> 数据库：MySQL 9.4.0，字符集 utf8mb4
> 导出时间：2026-05-21
> 数据时间跨度：2025-11 至 2026-05（约6个月运行数据）

---

## 一、概述

本数据集来自一套实际运行的**交通信号智能调控系统**，覆盖成都地区3个真实路口。系统实现了从摄像头车流采集、AI预测、配时优化到信号下发的完整闭环，同时集成了SUMO交通仿真能力。

核心业务流程：

```
摄像头采集车流 → traffic_flow_record
       ↓
AI预测排队/流量  → prediction_record
       ↓
生成配时计划    → batch_plan / analyze_result
       ↓
微调执行配时    → run_timing_adjustment
       ↓
信号相位下发    → phase_green_event / cycle_time
```


## 六、数据特点总结

### 6.1 数据采集

- **摄像头采集**：通过IP地址区分不同摄像头，每个摄像头绑定一个相位方向
- **双区域检测**：每个摄像头画面分为 area1 和 area2 两个检测区域，分别统计车辆数
- **采集频率**：秒级采集，数据量极大（1亿+条流量记录）

### 6.2 AI调控闭环

系统实现了完整的AI感知-决策-执行闭环：

1. **感知**：摄像头实时采集车流数据（traffic_flow_record）
2. **预测**：AI模型预测各方向车流和排队长度（prediction_record）
3. **决策**：AI分析各相位最优配时（analyze_result）
4. **计划**：生成信号配时执行计划（batch_plan）
5. **微调**：对基础配时进行AI微调（run_timing_adjustment）
6. **执行**：信号灯按计划切换（phase_green_event）
7. **反馈**：记录实际运行时长，形成闭环

### 6.3 SUMO仿真集成

- `crossing.sumo_id` 字段将真实路口映射到SUMO仿真路网节点
- `crossing_phase.sumo_val` 字段建立真实相位值与SUMO仿真相位值的映射
- `run_info.run_mode=3` 代表SUMO仿真模式

### 6.4 绿波带协调

- 支持多路口之间的绿波/红波协调控制
- 通过 offset 字段控制相邻路口间的相位差
- 支持硬切、软切、普通三种介入模式

### 6.5 数据无触发器/存储过程

所有业务逻辑在应用层实现，数据库层无触发器、存储过程、函数或视图，便于与仿真系统对接和数据迁移。
